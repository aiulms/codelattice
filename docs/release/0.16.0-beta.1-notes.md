# CodeLattice 0.16.0-beta.1 Release Notes

Date: 2026-05-22

Product version: `0.16.0-beta.1`

Status: external beta / daily-use candidate

## What This Release Is

`0.16.0-beta.1` refreshes the downloadable beta artifact so it matches current CodeLattice master. It includes the language, MCP, WebUI, workspace, and native governance work that landed after `v0.15.0-beta.1`.

This release remains static-analysis only. CodeLattice does not compile, run tests, execute project scripts, prove runtime reachability, or prove deletion safety.

## Headline Changes

- Default MCP surface is now AI-friendly: six facade entry tools in normal `ai` mode.
- Full expert/debug MCP surface remains available through `CODELATTICE_MCP_TOOLSET=full` and exposes the complete 49-tool profile.
- JavaScript support is included in the release artifact: `.js`, `.jsx`, `.mjs`, `.cjs`, ESM/CommonJS imports, package entry points, framework hints, and public-surface candidates.
- Shell support is included in the release artifact: shell functions, source edges, command calls, environment variables, and risky-script diagnostics.
- Workspace intelligence is included: workspace inventory, cross-project graph, cross-project impact, workspace insights, and native detect-changes context.
- WebUI workbench is included in source checkout workflows: project picker, bilingual UI, AntV G6 graph view, workspace dashboard, snapshot library, and live MCP runner.
- Native daily governance is now CodeLattice-native: `codelattice detect-changes` and `scripts/codelattice-precommit-check.sh`.

## MCP Toolsets

Default client mode:

```text

```

Visible tools: `codelattice_workflow`, `codelattice_project`, `codelattice_symbol`, `codelattice_change_review`, `codelattice_workspace`, `codelattice_cache`.

Debug/regression mode:

```text
CODELATTICE_MCP_TOOLSET=full
```

Visible tools: full 49-tool profile.

## Install

After the GitCode Release page is published, install from the release tarball:

```bash
export CODELATTICE_TOOL_DIR="$HOME/.local/share/codelattice-tool"
tmp_dir="$(mktemp -d /tmp/codelattice-install-XXXXXX)"
git clone --depth 1 https://gitcode.com/aiulms/codelattice.git "$tmp_dir"
bash "$tmp_dir/scripts/install-release.sh" \
  --version v0.16.0-beta.1 \
  --install-dir "$CODELATTICE_TOOL_DIR"
"$CODELATTICE_TOOL_DIR/codelattice-mcp.sh" --self-test
```

## Build From Source

```bash
git clone https://gitcode.com/aiulms/codelattice.git
cd codelattice
bash scripts/install-mcp.sh --build
```

## Package and Smoke


## Known Limitations

- Static results are investigation leads, not compiler/runtime/coverage proof.
- Dead-code and reachability signals are not deletion proof.
- JavaScript dynamic import/require is diagnostic-only.
- Shell analysis never executes scripts and does not replace shellcheck.
- C/C++ analysis does not do full preprocessing, template instantiation, overload resolution, or virtual dispatch resolution.
- Python analysis does not execute Python, install dependencies, inspect virtual environments, or infer dynamic types.
- WebUI runner binds only to `127.0.0.1` and stores local snapshots under `.codelattice-webui/`.
