# CodeLattice 0.17.0-beta.1 Release Notes

Date: 2026-06-05

Product version: `0.17.0-beta.1`

Status: external beta / daily-use candidate

## What This Release Is

`0.17.0-beta.1` refreshes the downloadable beta artifact after the MCP AI usability and workflow hardening work that landed after `v0.16.0-beta.1`.

This release remains static-analysis only. CodeLattice does not compile, run tests, execute target project scripts, prove runtime reachability, or prove deletion safety.

## Headline Changes

- AI-facing MCP workflow is significantly more usable: `codelattice_workflow(mode=ask)` now returns structured evidence, confidence, static-only guidance, read order, call-chain orchestration, diagnose routing, and what-if previews.
- Default MCP surface remains stable and small: 6 facade-first tools in `CODELATTICE_MCP_TOOLSET=ai`; full debug/regression mode remains 49 tools.
- Compact responses are tuned for AI token budgets: facade outputs bound root/source-only diagnostics, trim repeated schedule/analysis semantics, and preserve concise navigation hints.
- Job runtime is easier for agents: project, workspace, symbol, and change-review job modes support `wait=true` / `timeoutMs`, paged detail, structured timeout recovery, and job control-plane calls without redundant root validation.
- Root and language routing is safer: single-project `language=auto` paths resolve to a concrete language before cache probing, job submission, and facade output wrapping.
- TypeScript/JavaScript and delta-analysis ergonomics improved with stale-first cache behavior, TS delta symbol extraction, camelCase tokenization fixes, and compact workflow payload budgets.
- Installed/dev acceptance now checks real MCP JSON-RPC sessions, compact payload sizes, tool counts, call-chain shape, ask v2 behavior, cache status, and promoted-runtime freshness.

## MCP Toolsets

Default client mode:

```text
CODELATTICE_MCP_TOOLSET=ai
```

Visible tools: `codelattice_workflow`, `codelattice_project`, `codelattice_symbol`, `codelattice_change_review`, `codelattice_workspace`, `codelattice_cache`.

Debug/regression mode:

```text
CODELATTICE_MCP_TOOLSET=full
```

Visible tools: full 49-tool profile.

## Install

After the GitCode Release page is published, install from the release tarball:

```bash
export CODELATTICE_TOOL_DIR="$HOME/.local/share/codelattice-tool"
tmp_dir="$(mktemp -d /tmp/codelattice-install-XXXXXX)"
git clone --depth 1 https://gitcode.com/aiulms/codelattice.git "$tmp_dir"
bash "$tmp_dir/scripts/install-release.sh" \
  --version v0.17.0-beta.1 \
  --install-dir "$CODELATTICE_TOOL_DIR"
"$CODELATTICE_TOOL_DIR/codelattice-mcp.sh" --self-test
```

## Build From Source

```bash
git clone https://gitcode.com/aiulms/codelattice.git
cd codelattice
bash scripts/install-mcp.sh --build
```

## Known Limitations

- Static results are investigation leads, not compiler/runtime/coverage proof.
- Dead-code and reachability signals are not deletion proof.
- Ask/workflow recommendations still require human or runtime verification before production changes.
- JavaScript dynamic import/require is diagnostic-only.
- Shell analysis never executes scripts and does not replace shellcheck.
- C/C++ analysis does not do full preprocessing, template instantiation, overload resolution, or virtual dispatch resolution.
- Python analysis does not execute Python, install dependencies, inspect virtual environments, or infer dynamic types.
- WebUI runner binds only to `127.0.0.1` and stores local snapshots under `.codelattice-webui/`.
