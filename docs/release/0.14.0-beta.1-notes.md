# CodeLattice 0.14.0-beta.1 Release Notes

Date: 2026-05-16

## Summary

`0.14.0-beta.1` is a release beta hardening pack. It does not add another language line; it packages and documents the multi-language work already landed so external beta users can install, self-test, analyze fixtures, and connect the MCP sidecar with a shorter path.

Product version: `0.14.0-beta.1`
MCP serverVersion: `0.13.0`

The product version tracks release artifacts. MCP `serverVersion` tracks the sidecar tool/profile contract and may advance independently.

## Who Should Try This Beta

Try this beta if you:

- Work on large, legacy, or tangled codebases and want a read-first / review-first graph before changing code.
- Want local-only code intelligence for AI coding assistants without uploading source.
- Need static symbol, import/include, call, impact, and quality-gate context through CLI or MCP.
- Are evaluating Rust / Cangjie / ArkTS / TypeScript / C / C++ / Python local graph analysis.

Do not treat this as GA, a compiler, an IDE, a language server, or a complete static analyzer.

## Installation

From a source checkout:

```bash
git clone https://gitcode.com/aiulms/codelattice.git
cd codelattice
bash scripts/install-mcp.sh --build
scripts/codelattice-mcp.sh --self-test
```

Promote to a stable local wrapper path:

```bash
export CODELATTICE_TOOL_DIR="$HOME/.local/share/codelattice-tool"
scripts/promote-to-local-tool.sh --install-dir "$CODELATTICE_TOOL_DIR"
"$CODELATTICE_TOOL_DIR/codelattice-mcp.sh" --self-test
```

After the GitCode Release page is published, install from the release tarball:

```bash
bash scripts/install-release.sh \
  --version v0.14.0-beta.1 \
  --install-dir "$CODELATTICE_TOOL_DIR"
```

The installer downloads the tarball, verifies `.sha256`, installs the stable MCP wrapper, and runs self-test. It does not modify Codex, opencode, Claude, or shell configuration files.

## Known Limitations

- Not a compiler, IDE, language server, hosted upload service, or GitNexus-RC replacement.
- No full type inference or trait solving.
- No macro expansion.
- No full C/C++ preprocessor.
- No target project code execution.
- No hosted upload/index path.
- Phase A language lines should be validated on user projects before relying on exact graph counts.

## Upgrade Notes

- `0.13.0-beta.2` remains the previous all-language packaging correction for Rust/Cangjie/ArkTS/TypeScript.
- `0.14.0-beta.1` is the recommended beta for the full seven-language artifact and real-project baseline evidence.
- Re-run `codelattice-mcp.sh --self-test` after upgrading.
- If using a stable wrapper, re-run `promote-to-local-tool.sh --install-dir <path>` after building or installing the new version.
- Do not reuse stale AI client config paths; prefer `/path/to/CodeLattice-Tool/codelattice-mcp.sh`.
